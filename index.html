Dylan Dog Citation LOD Project

A project to create a linked open data (LOD) representation of Dylan Dog citations.

By Alessandro Rocchi

Introduction

The goal of this project is to create structured data starting from a Wikipedia page and related entities

Domain Analysis

Project Idea

The main idea behind this project is to leverage the wealth of information available on Wikipedia and other online sources to create a comprehensive and interconnected dataset of Dylan Dog citations. The research question is: How can we effectively represent and link Dylan Dog citations in a structured, machine-readable format?

The first step in this process is to identify and extract relevant information from the Wikipedia page and other sources. I choose to focus on the italian wikipedia page Dylan Dog, which refers to the fictional character created by Tiziano sclavi.

This page was choosen instead of the one related to the comic series because it provides a more focused view on the character and an exaustive section for citations and references.

Entities

After choosing the page I identify a total of 27 entities which was been mentionated in the wikipedia-page. Every entity has been linked to their authority control, where was applicable. The main authority control used was VIAF

You can find all the entities regrouped in a table and divided in categories:

  • Persons (Real or Fictional)
  • Organizations
  • Bibliographic sources
  • Locations

The table has been generate by using a XSLT to HTML transformation using javascript. In this way if we add entities to the XML/TEI file they will automatically been added to this table. (Note: it's a table dynamically generated and use XSLTProcessor so if it's not working, please check the browser console for errors.)

Knowledge organization: elaborate models

Theoretical Model

After identifying the key entities and their relationships, I developed a theoretical model to represent the knowledge structure of Dylan Dog citations. This model aims to capture the semantic connections between different types of entities. And to show connections which weren't explicitly mentioned in the original text.

The model is composed of three main components: the main entities, their attributes, and the external resources (URIs) used to link them to authoritative sources or catalogues. In particular, the autoritative sources used are: VIAF and OPAC SBN. Wikidata has been used to conciliate and a semantic disambiguation for entities which are not bibliographic. For the location has been used Geonames as authoritative source.

The graph was created using yED software and it was included a zoom javascript plugin to enable interactive exploration and make it more readble. On the left you can find a legend which explain the symbols used in the graph.

Conceptual Model

After realising the theoretical model, I developed a conceptual model to create a formal representation of the theoretical model. I choose to reuse exiting ontologies where possible and create a personal ontology to fill the gaps with existing ones.

The conceptual model is based on the following ontologies:

Project Ontologies:

Prefix: DD

DDOntology

Why I chose it: The custom ontology I created to fill the gaps left by existing ontologies. It defines specific classes and properties for Dylan Dog citations, ensuring that the dataset accurately reflects the unique aspects of the domain.

URI: https://alessandro-rocchi.github.io/project_InformationScienceandCH/ontology/DDOntology#

Prefix: FRBRoo

FRBR object-oriented

Why I chose it: Crucial for modeling the complex bibliographic hierarchy. I used it to logically separate the conceptual work from its physical expressions and publications (e.g., monographs vs. series).

URI: http://iflastandards.info/ns/fr/frbr/frbroo/

Official Docs

Prefix: FOAF

Friend of a Friend

Why I chose it: The absolute standard for describing people. I employed it to model authors and actors, and to create links between physical entities and their visual depictions.

URI: http://xmlns.com/foaf/0.1/

Official Docs

Prefix: Schema.org

Schema.org

Why I chose it: A universal vocabulary supported by major search engines. Useful for mapping basic cross-sectional properties, such as work titles, publication dates, and structural relations.

URI: http://schema.org/

Official Docs

Prefix: dcterms

Dublin Core Terms

Why I chose it: The universal standard for basic metadata. It provides highly interoperable properties for fundamental descriptive elements like titles, dates, and creators of the comic books.

URI: http://purl.org/dc/terms/

Official Docs

Prefix: cbo

Comic Book Ontology

Why I chose it: The most specific and semantic vocabulary for the comic book domain. It allowed me to accurately define complex publishing concepts like issue numbers and comic series.

URI: http://comicmeta.org/cbo

Official Docs

Prefix: dbo

DBpedia Ontology

Why I chose it: Extracted directly from Wikipedia, it serves as a massive Linked Open Data hub. Useful for cross-referencing real-world entities (authors, places) to enrich the dataset.

URI: http://dbpedia.org/ontology/

Official Docs

Prefix: rdfs

RDF Schema

Why I chose it: It provides the structural building blocks for the ontology. Essential for defining class hierarchies (subclasses) and adding human-readable labels to our custom entities.

URI: http://www.w3.org/2000/01/rdf-schema#

Official Docs

Prefix: crm

CIDOC CRM

Why I chose it: The reference model for cultural heritage. It is perfect for describing historical events, physical creation processes, and the lifecycle of cultural artifacts over time.

URI: http://cidoc-crm.org/cidoc-crm/

Official Docs

Prefix: rdf

RDF Syntax

Why I chose it: The foundational data model for the Semantic Web. It provides the core syntax required to define resources and structure information as a directed graph.

URI: http://www.w3.org/1999/02/22-rdf-syntax-ns#

Official Docs

Prefix: owl

Web Ontology Language

Why I chose it: It adds advanced computational logic. Crucial for defining identity alignments across different datasets and setting strict cardinality restrictions for the Reasoner.

URI: http://www.w3.org/2002/07/owl#
