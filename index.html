Dylan Dog Citation LOD Project

A project to create a linked open data (LOD) representation of Dylan Dog citations.

By Alessandro Rocchi

Introduction

The goal of this project is to create structured data starting from a Wikipedia page and related entities

Domain Analysis

Project Idea

The main idea behind this project is to leverage the wealth of information available on Wikipedia and other online sources to create a comprehensive and interconnected dataset of Dylan Dog citations. The research question is: How can we effectively represent and link Dylan Dog citations in a structured, machine-readable format?

The first step in this process is to identify and extract relevant information from the Wikipedia page and other sources. I choose to focus on the italian wikipedia page Dylan Dog, which refers to the fictional character created by Tiziano sclavi.

This page was choosen instead of the one related to the comic series because it provides a more focused view on the character and an exaustive section for citations and references.

Entities

After choosing the page I identify a total of 27 entities which was been mentionated in the wikipedia-page. Every entity has been linked to their authority control, where was applicable. The main authority control used was VIAF

You can find all the entities regrouped in a table and divided in categories:

  • Persons (Real or Fictional)
  • Organizations
  • Bibliographic sources
  • Locations

The table has been generate by using a XSLT to HTML transformation using javascript. In this way if we add entities to the XML/TEI file they will automatically been added to this table.

Knowledge organization: elaborate models

Theoretical Model

After identifying the key entities and their relationships, I developed a theoretical model to represent the knowledge structure of Dylan Dog citations. This model aims to capture the semantic connections between different types of entities. And to show connections which weren't explicitly mentioned in the original text.

The model is composed of three main components: the main entities, their attributes, and the external resources (URIs) used to link them to authoritative sources or catalogues. In particular, the autoritative sources used are: VIAF and OPAC SBN. Wikidata has been used to conciliate and a semantic disambiguation for entities which are not bibliographic. For the location has been used Geonames as authoritative source.

The graph was created using yED software and it was included a zoom javascript plugin to enable interactive exploration and make it more readble. On the left you can find a legend which explain the symbols used in the graph.
